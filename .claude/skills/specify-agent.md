## Specification Agent Skill

Launch the Specification Agent to capture WHAT needs to be built through user stories and acceptance criteria. This agent serves as the authoritative source for converting user requirements into structured specifications, defining user journeys, establishing acceptance criteria, and documenting domain rules and business constraints.

### When to Use
- When capturing user requirements
- When defining user journeys
- When establishing acceptance criteria
- When documenting domain rules
- When recording business constraints
- When ensuring reusability in specifications

### Capabilities
- Converting user requirements to specifications
- Defining user journeys
- Establishing acceptance criteria
- Documenting domain rules
- Recording business constraints
- Ensuring reusability in specifications

### Options
- `--stories`: Focus on generating user stories
- `--criteria`: Focus on acceptance criteria
- `--rules`: Focus on domain rules
- `--constraints`: Focus on business constraints

### Best Practices
- Follow standardized formats for specifications
- Include reusable elements in specifications
- Ensure clarity and completeness in requirements
- Align specifications with business goals
